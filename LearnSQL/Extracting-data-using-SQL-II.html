    Date Warehouse:

    Database vs Datawarehouse

  • Both have tables and both have data.
  • Database will have real time data (Live users).
  • There are some cases where we dont want real time data (Historical Data). Primary purpose is to do analysis
  • Datawarehouse - If we want to do any kind of analysis. They do a copy of data from Database.
  • Warehouse - Storage. We just move the data on which we want to do some analysis.
  • Warehouse is optimized for data analysis
  • Base is optimized for giving real time data.
  • Amazon Redshift, Big Query
  • Data from different databases can be combined into a single datawarehouse and combine to run some analytics
  • Warehouses can hold more data.

    • Writing SQL Query:

  • We can have tables without Primary/Candidate Keys
  • SELECT column_name FROM folder_name.table_name; to retrieve single column data from datawarehouse.
  • SELECT column_name FROM table_name; to retrieve single column data from database(mysql).
  • SELECT column1_name,column2_name,column3_name FROM folder_name.table_name; to retrieve multiple column data from datawarehouse.
  • SELECT column1_name,column2_name,column3_name FROM table_name; to retrieve multiple column data from database(mysql).
  • SELECT * FROM folder_name.table_name; to retrieve all columns data from datawarehouse.
  • SELECT * FROM table_name; to retrieve all columns data from database(mysql).
  • # for comments in Big Query
  • /**/ for comments in mysql
  • Columns can be repeated in SQL Queries.
