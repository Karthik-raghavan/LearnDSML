Revisiting Datatypes

  • Only in the case if division, there are times when the resulting number is float.

    • Boolean:

  • True or False values.
  • These words are also Keywords.
  • There will be no quotation marks on True or False
  • True/TRUE/true are all valid.

    • String - Text Data:

  • CHAR - Allocates exactly the exact length provided. Memory wastage is there.
  • VARCHAR - Reserves only the required length. No memory wastage.
  • CHAR vs VARCHAR - CHAR is better than VARCHAR because of performance aspectss.
  • If we clearly know, how many characters a string is going to take, then use CHAR.
  • KEYWORDS - Words which SQL already understands.
  • These words have special meaning.
  • Example - SELECT, FROM
  • In SQL, Keywords are not case sensitive.(meaning we can use lower case letters.)

    • Keys in SQL:

  • PRIMARY KEY-Value that helps us unlock one row at a time.
  • There should be only 1 Primary Key.
  • NULL VALUES are not values, it is the absence/missing of values. NULL Values are not a unique Category.
  • PRIMARY KEYS are UNIQUE, NOT NULL.
  • UNIQUE KEYS will have NULL Values but all of them are Unique(Non Repeating).
  • PRIMARY vs UNIQUE - Primary will not have missing values but Unique can have unique values.
  • We should not compare Null Values.
  • CANDIDATE KEYS-If there are multiple KEYS which can be PRIMARY KEYS, They are all called as CANDIDATE KEYS and The key which is chosen as PRIMARY KEY is a choice and do not have any conditions.
  • FOREIGN KEY A key which is a Primary key of another table which is used in the current table.
  • Foreign Keys can be Duplicate and has to be in reference to the Primary Key.
  • Foreign keys must be of the same data type as the primary key they reference.
  • Foreign keys can also have missing values.
  • Foreign keys are optional.
  • Multiple Foreign keys are also possible.
